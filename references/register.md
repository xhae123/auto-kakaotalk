# 상대 등록 흐름

## 이 문서의 역할

등록은 단순 스크립트 호출이 아니라 **사용자와의 톤 calibration 대화**다. 루프는 매 메시지마다 사용자에게 묻지 않으므로 (자동 전송이 기본), calibration 이 **유일하게 사용자가 개입하는 자리**다. 여기서 합의된 뉘앙스가 persona.md 에 박히고, 이후 AI 는 그걸 따른다.

## 절차

### 1. 방 찾기

```bash
bash scripts/adapters/kakao.sh resolve | jq '.[] | {chat_id, display_name}'
```

현 helper 는 display_name 기준 매칭이므로 실질 키는 이름. 정확 일치 없으면 사용자에게 후보 확정.

### 2. register.py 실행 (과거 대화 수집)

```bash
python3 scripts/register.py \
  --chat-id "<이름>" \
  --display-name "<이름>" \
  --history-limit 500
```

스크립트는 DB 백필 + 빈 persona.md 생성까지만. LLM 은 안 호출.

### 3. 사회과학적 분석 리포트 (Claude 가 직접)

여기가 핵심. Claude 가 DB 의 과거 대화를 읽고 **사용자에게 관계와 말투를 분석해 보고**한다.

```bash
python3 scripts/db.py get-context --chat-id <이름> --limit 200
```

리포트 템플릿 (길이는 상황 따라 조절):

```
[용진 분석 리포트]

▸ 관계의 성격
  - 친밀도: 반말, 높은 욕설 빈도, 자조 공유 → 10년 이상 친구로 추정
  - 주도권: 메시지 시작 비율 58:42 (용진:나). 용진이 살짝 더 자주 거는 편
  - 응답 리듬: 평균 응답 간격 나 12분 / 용진 4분. 내가 늦는 편

▸ 말투 패턴 (나의)
  - 평균 길이 1.4문장. 짧음
  - 말끝: ~임/~ㅇㅇ/~ㄱㄱ 등 축약형 우세
  - 이모지 거의 없음 (사용률 3%)
  - 욕설: 맥락적으로 섞음, 친밀감 표시로 사용
  - 존댓말 0%

▸ 최근 변화
  - 최근 2주 내 응답 지연 평균 38분으로 증가
  - "좆됏군", "씨팔련이" 등 톤이 격해짐 — 이 친구와 최근 긴장?

▸ 답장하지 말아야 할 패턴 (추정)
  - 계좌/송금 관련
  - "꿈" 에 대한 긴 서사 (과거 대화에서 사용자가 안 반응한 이력)

이 분석 맞아? 내가 대신 답할 때 어떤 뉘앙스로 가는 게 좋을까?
```

### 4. 사용자와 조율 (필수)

리포트를 본 사용자가 교정 / 추가 지시:

- "친밀도 분석은 맞는데 최근 톤 격해진 건 그냥 장난임. 신경 쓰지 마"
- "용진한텐 답장 좀 늦게 하는 편이 자연스러워. 꾸며내서 바로 답하지 마"
- "'형' 이라는 호칭 쓰는 거 누락됐어. 그거 반영해"
- "계좌 얘기는 정말 skip 해줘"

사용자의 교정을 반영해 `state/personas/<chat_id>.md` 에 **Claude 가 직접 Write 도구로** 작성:

```md
# 용진 용 지시사항 (calibration 2026-04-23)

## 관계
- 10년 이상 친구. 형이라고 부름.
- 친밀도 높음. 욕설 섞인 농담은 자연스러움.

## 말투
- 짧게. 평균 1-2문장.
- ~임/~ㅇㅇ/~ㄱㄱ 축약형.
- 이모지 거의 쓰지 않음.

## 주의
- 계좌/송금 얘기 나오면 skip.
- "꿈" 얘기 시작하면 짧게 "ㅇㅇ" 정도만. 길게 공감 금지.
- 답장 리듬은 원래 12분 평균. 너무 즉답하지 말 것.
```

### 5. 최종 확인

```
이렇게 잡았어. 이 사람한텐 이 뉘앙스로 갈게. 시작해도 돼?
```

사용자 "ㅇㅇ" → 등록 완료. 이후 루프는 이 persona.md 를 매 cycle 에 읽고, 사용자에게 더는 묻지 않는다.

## 왜 calibration 이 필요한가

루프는 사용자에게 매 메시지마다 안 묻는다. 묻지 않으려면 **무엇을 따를지 한 번은 정확히 합의**해야 한다. 대충 잡아두고 자동 발사하면, 사용자가 "내가 저런 말 안 했는데" 로 신뢰를 잃는다. 한 번 잃으면 복구 안 됨.

그래서 calibration 은 **유일한 필수 승인 포인트**다. 이후의 자동화는 이 합의에서 나온다. 사용자가 calibration 을 건너뛰려 하면 Claude 는 막는다 — "이 상대는 아직 내가 어떻게 답할지 안 정해졌어. 먼저 분석 같이 볼래?"

## 기존 상대 재-calibration

상대와의 관계가 변했거나 (친해졌거나 멀어졌거나), persona 가 몇 달 된 경우 재-calibration. 명령:

```
"용진 다시 분석해줘"
```

Claude 는 최근 대화 위주로 재리포트 → 사용자와 조율 → persona.md 업데이트.

## 실패 모드

- **resolve 가 빈 목록**: 어댑터 auth 실패. `cycle.sh check` 로 점검.
- **history 가 빈 배열**: 방에 메시지가 없음. calibration 불가능 — 등록 보류하거나 사용자가 직접 persona.md 를 작성.
- **own_messages 가 0**: helper 필드명 변경 의심. 중단하고 디버깅.

## 불변 조건

- 같은 chat_id 로 register.py 재실행해도 기존 persona.md 는 덮어쓰지 않는다.
- calibration 없이 등록된 상대는 `§Drafting 원칙` 에 따라 skip 된다 (persona.md 가 비면 보내지 않음).
- 백필된 과거 대화는 `state='learned'` — `list-pending` 에 안 걸림.
